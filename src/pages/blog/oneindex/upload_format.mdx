# 共享网盘上传说明

出于方便群友的使用的考虑，为向共享网盘上传资源做出了一些说明：

## 上传方式

共享网盘的 OneDrive 账号由所有的管理员持有。如果有群友想要上传资源可以联系管理员索要账号密码；为了共享网盘能够持久的服务群友，得到账号密码的群友也不要将它随便地透露出去。

得到账号后，可以通过[网页端](https://office.com)的 OneDrive 登陆上传或直接将共享 OneDrive 账户增加到本地，使用本地的客户端上传。虽然客户端上传不支持断点续传，以 OneDrive 现在国内的访问速度要想上传什么就得一直让电脑挂机，但是监狱目前网页版 OneDrive 国内访问速度感人，故推荐使用本地客户端进行上传。

## 压缩参数

虽然搬运资源非常的方便，但是推荐群友都自己重新压缩后上传；推荐的压缩参数如下：

| 项目       | 对应的说明                             |
| ---------- | -------------------------------------- |
| 分卷压缩   | 分卷大小不超过 2GB                     |
| 恢复记录   | RAR，设置 3% 的恢复记录（使用 WinRAR） |
| 解压密码   | `aoikaze`，如果有内层压缩包则密码统一  |
| 上传者标识 | 为压缩文件添加注释并注明               |

下面是我们采用其中的一些标准的原因：

- **为什么要采用分卷并限制分卷的大小？**   
  OneDrive 对于每一个开启的下载窗口有时间限制；目前 OneDrive 在国内的速度并不快，分卷过大或不分卷可能会导致下载尚未完成而下载窗口关闭，导致无法下载的情况。

- **为什么要使用恢复记录？**  
  恢复记录可以避免一定程度内的压缩文件的出错—— 如果发生了错误，则可以使用恢复记录进行一定程度的修复而避免重新下载；而类似 OneDrive 的不稳定的网络下载可能会引发这种错误。

- **为什么要重新压缩？**  
  虽然目前使用的共享网盘是 OneDrive，但是不排除之后使用国内网盘的可能性：而国内网盘会通过文件的特征值来排除一个文件和它所有的复制；重新压缩得到的文件的文件特征值不同，可以避免国内的网盘的封杀。

> 此外，对于存档在游戏根目录下的场合，自压时可能需要先清除存档。
