# 压缩参数

虽然搬运资源非常的方便，但是推荐群友都自己重新压缩后上传；对于不经常自己压缩文件的群友，我们推荐的压缩参数如下：

| 项目       | 对应的说明                             |
| ---------- | -------------------------------------- |
| 分卷压缩   | 分卷大小不超过 2GB                     |
| 恢复记录   | RAR，设置 3%~10% 的恢复记录（使用 WinRAR） |
| 解压密码   | `aoikaze` 或不设置，如果有内层压缩包则密码统一 |
| 上传者标识 | 为压缩文件添加注释并注明               |
| 测试 | 压缩完成后建议自行测试压缩文件没有错误后再上传 |

下面是我们采用其中的一些标准的原因：

- **为什么要采用分卷并限制分卷的大小？**   
  OneDrive 对于每一个开启的下载窗口有时间限制；目前 OneDrive 在国内的速度并不快，分卷过大或不分卷可能会导致下载尚未完成而下载窗口关闭，导致无法下载的情况。

- **为什么要使用恢复记录？**  
  恢复记录可以避免一定程度内的压缩文件的出错—— 如果发生了错误，则可以使用恢复记录进行一定程度的修复而避免重新下载；而类似 OneDrive 的不稳定的网络下载可能会引发这种错误。

- **为什么要重新压缩？**  
  虽然目前使用的共享网盘是 OneDrive，但是不排除之后使用国内网盘的可能性：而国内网盘会通过文件的特征值来排除一个文件和它所有的复制；重新压缩得到的文件的文件特征值不同，可以避免国内的网盘的封杀。

> 此外，对于存档在游戏根目录下的场合，自压时可能需要先清除存档。

## 简明教程

如果你不会设置 WinRAR 压缩参数，那么下面的说明可能对你有用：

再你打开右键压缩菜单之后，可以按照以下的指引完成指定参数的设置。

![step1](/image/post/oneindex/archive1.png)

关于加密文件名：设置后需要输入密码才能看到注释，但是可以为之后可能的上传其他网盘做准备；不做要求。

![step2](/image/post/oneindex/archive2.png)

设置完分卷、密码和恢复记录之后，再设置压缩注释；比如：

```
解压密码：aoikaze
from Nanoah.
```

注释内容需表示上传者。

![step3](/image/post/oneindex/archive3.png)

如果你愿意经常上传文件，那么建议你创建一个 txt 文件，每次压缩时都从文件加载注释。
